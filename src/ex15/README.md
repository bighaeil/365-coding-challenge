# ex15. 프린터 우선순위 (Printer Priority)

## 문제 설명

일반적인 프린터는 인쇄 요청이 들어온 순서대로 인쇄합니다.
하지만 이 프린터는 다음 규칙으로 동작합니다.

1. 인쇄 대기목록의 **가장 앞**에 있는 문서를 꺼냅니다.
2. 대기목록에서 꺼낸 문서보다 **우선순위가 높은 문서**가 있으면, 꺼낸 문서를 대기목록 **맨 뒤**에 추가합니다.
3. 그런 문서가 없다면 꺼낸 문서를 **인쇄**합니다.

인쇄 대기목록 `priorities`와 내가 요청한 문서의 위치 `location`이 주어질 때,
내 문서가 **몇 번째로 인쇄되는지** 반환하세요.

> 카카오 코딩테스트 실제 출제 유형입니다.

## 입력 조건

- `priorities`의 길이는 `1` 이상 `100` 이하입니다.
- 우선순위는 `1` 이상 `9` 이하입니다. 숫자가 클수록 우선순위가 높습니다.
- `location`은 `0` 이상 `priorities.length - 1` 이하입니다.

## 예제

**예제 1**
```
입력: priorities = [2, 1, 3, 2], location = 2
출력: 1
설명: 우선순위가 3인 문서(인덱스 2)가 가장 먼저 인쇄됩니다.
```

**예제 2**
```
입력: priorities = [1, 1, 9, 1, 1, 1], location = 0
출력: 5
설명: 우선순위 9인 문서가 먼저 나가고, 나머지는 순서대로 인쇄.
     인덱스 0 문서는 5번째로 인쇄됩니다.
```

## 풀이 아이디어

### 핵심 아이디어
큐(Queue)로 대기목록을 관리하고, 각 문서에 **원래 인덱스**를 함께 저장해 추적합니다.

### 방법. 큐 시뮬레이션 — O(n²)
1. `[우선순위, 원래인덱스]` 쌍으로 큐 초기화
2. 큐에서 문서를 꺼냄
3. 남은 큐에 더 높은 우선순위가 있으면 → 꺼낸 문서를 큐 맨 뒤로
4. 없으면 → 인쇄 카운트 +1, 꺼낸 문서가 내 문서면 카운트 반환

```
priorities = [2, 1, 3, 2], location = 2

초기 큐: [[2,0], [1,1], [3,2], [2,3]]

1회: [2,0] 꺼냄 → 더 높은 우선순위(3) 있음 → 맨 뒤로
     큐: [[1,1], [3,2], [2,3], [2,0]]

2회: [1,1] 꺼냄 → 더 높은 우선순위(3) 있음 → 맨 뒤로
     큐: [[3,2], [2,3], [2,0], [1,1]]

3회: [3,2] 꺼냄 → 더 높은 우선순위 없음 → 인쇄! count=1
     인덱스 2 == location(2) → 1 반환!
```

## 면접 팁

- **"왜 큐를 사용하나요?"** → 문서를 맨 앞에서 꺼내고 맨 뒤에 추가하는 FIFO 구조가 필요하기 때문입니다. 배열의 `shift()`는 O(n)이지만 간단한 구현에서는 허용됩니다.
- **"시간 복잡도는?"** → O(n²). 매 turn마다 큐 전체를 순회해 최대 우선순위를 확인하고, 이를 최대 n번 반복합니다.
- **"원래 인덱스를 함께 저장하는 이유는?"** → 문서가 큐를 순환하면서 위치가 바뀌기 때문에 내 문서를 추적하려면 원래 인덱스를 함께 보관해야 합니다.
- **카카오 실전 팁**: 시뮬레이션 문제는 문제의 규칙을 코드로 그대로 옮기는 것이 핵심입니다. 규칙을 명확히 파악한 후 자료구조(여기선 큐)를 선택하세요.
