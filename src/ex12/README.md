# ex12. 연결 리스트 사이클 감지 (Linked List Cycle Detection)

## 문제 설명

단일 연결 리스트(singly linked list)의 **head 노드**가 매개변수로 주어집니다. 각 노드는 다음과 같은 형태입니다.

```js
// ListNode
{ val: number, next: ListNode | null }
```

`next` 포인터를 따라 리스트를 순회할 때, 이전에 방문한 노드로 되돌아가는 **사이클(cycle)**이 존재하면 `true`, 끝(`null`)에 도달하면 `false`를 return 하도록 `solution` 함수를 완성해 주세요.

> 사이클이란 어떤 노드의 `next` 포인터가 이전에 방문한 노드를 가리켜, 무한 루프를 형성하는 상태를 말합니다.
> **주의**: 풀이 함수는 head 노드만 받습니다. 노드에는 인덱스나 방문 여부 같은 정보가 없으므로, 포인터를 실제로 순회하며 판별해야 합니다.

```js
function solution(head) {
  // head부터 next를 순회하며 사이클 존재 여부를 반환
  return answer; // boolean
}
```

> **추가 도전**: 사이클이 시작되는 노드(진입점)도 반환할 수 있나요?

## 제한사항

- 노드의 수는 `0` 이상 `10,000` 이하입니다. (빈 리스트면 `head === null`)
- 노드의 값(`val`)은 정수입니다.
- 리스트는 head에서 `next`로만 이동할 수 있는 단일 연결 리스트입니다.

## 입출력 예

> 아래 표의 `arr`, `pos`는 **테스트 하네스가 리스트를 만드는 방법**을 나타냅니다.
> `arr` 값으로 노드를 순서대로 잇고, `pos !== -1`이면 tail의 `next`를 `arr[pos]` 노드에 연결합니다.
> 풀이 함수에는 이렇게 만들어진 **head 노드**가 전달됩니다.

| arr (구성용) | pos (구성용) | solution(head) |
|-----|-----|--------|
| `[3, 2, 0, 4]` | `1` | `true` |
| `[1, 2]` | `0` | `true` |
| `[1]` | `-1` | `false` |

### 입출력 예 설명

**입출력 예 #1** — `arr=[3, 2, 0, 4], pos=1`
```
3 → 2 → 0 → 4
    ↑_______↑   (tail 노드 '4'의 next가 인덱스 1 노드 '2'를 가리킴)
```
`3→2→0→4→2→0→4→…`로 무한 반복되므로 사이클 존재 → `true`.

**입출력 예 #2** — `arr=[1, 2], pos=0`
```
1 → 2
↑___↑   (tail 노드 '2'의 next가 인덱스 0 노드 '1'을 가리킴)
```
`1→2→1→…`로 반복되므로 `true`.

**입출력 예 #3** — `arr=[1], pos=-1`
```
1 → 끝   (tail 노드의 next = null)
```
`null`(끝)에 도달하므로 사이클 없음 → `false`.

## 풀이 아이디어

head부터 `node = node.next`로 이동하며, `null`에 도달하면 사이클 없음입니다.

### 방법 1. 해시셋(Set) 활용 - O(n) 시간, O(n) 공간
- 방문한 **노드(참조)**를 Set에 저장
- 이미 Set에 있는 노드를 다시 만나면 → 사이클
- `null`(끝)에 도달하면 → 사이클 없음

### 방법 2. 플로이드 순환 감지 (Floyd's Cycle Detection) - O(n) 시간, **O(1) 공간** ⭐
- **느린 포인터(slow)**: `slow = slow.next` — 한 번에 1칸 이동
- **빠른 포인터(fast)**: `fast = fast.next.next` — 한 번에 2칸 이동
- 사이클이 있다면 두 포인터는 반드시 만남 (토끼와 거북이 알고리즘)
- fast가 `null`(끝)에 도달하면 사이클 없음

```
head = 3 → 2 → 0 → 4 ─┐
           ↑__________┘   (사이클)

slow(1칸): 3 → 2 → 0 → 4 → 2 → ...
fast(2칸): 3 → 0 → 2 → 0 → 2 → ...
                       ↑ 두 포인터가 만남!
```

## 면접 팁

- **"왜 플로이드 알고리즘인가?"**: 공간 복잡도 O(1)이 핵심. 해시셋은 O(n) 메모리가 필요하지만, 플로이드는 포인터 2개만 사용합니다.
- **"두 포인터가 반드시 만나는 이유?"**: 사이클 내에서 fast는 slow보다 1칸씩 더 빠르게 따라잡으므로, 사이클 길이 내에서 반드시 만납니다.
- **응용 질문**: "사이클 시작점을 어떻게 찾나요?" → 만난 지점에서 slow를 head로 리셋 후 둘 다 1칸씩 이동하면 사이클 시작점에서 다시 만납니다.
- **실무 연관성**: 메모리 누수 감지, 데드락 탐지, 무한 루프 방지 등에 응용됩니다.
