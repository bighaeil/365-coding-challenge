# ex12. 연결 리스트 사이클 감지 (Linked List Cycle Detection)

## 문제 설명

값 배열 `arr`과 사이클 위치 `pos`가 매개변수로 주어집니다.

- `arr`의 각 원소를 노드로 하여 순서대로 연결한 단일 연결 리스트(singly linked list)를 구성합니다.
- `pos`가 `-1`이 아니면 마지막(tail) 노드의 `next`가 `arr[pos]`에 해당하는 노드를 가리켜 **사이클(cycle)**을 형성합니다.

이때 리스트에 사이클이 존재하면 `true`, 존재하지 않으면 `false`를 return 하도록 `solution` 함수를 완성해 주세요.

> 사이클이란 어떤 노드의 `next` 포인터가 이전에 방문한 노드를 가리켜, 무한 루프를 형성하는 상태를 말합니다.

```js
function solution(arr, pos) {
  // 연결 리스트를 구성하고 사이클 존재 여부를 반환
  return answer; // boolean
}
```

> **추가 도전**: 사이클이 시작되는 노드(진입점)도 반환할 수 있나요?

## 제한사항

- `arr`의 길이(노드 수)는 `0` 이상 `10,000` 이하입니다.
- `arr`의 각 원소(노드 값)는 정수입니다.
- `pos`는 tail 노드의 `next`가 연결되는 노드의 인덱스입니다 (0-based). `-1`이면 사이클 없음.

## 입출력 예

| arr | pos | result |
|-----|-----|--------|
| `[3, 2, 0, 4]` | `1` | `true` |
| `[1, 2]` | `0` | `true` |
| `[1]` | `-1` | `false` |

### 입출력 예 설명

**입출력 예 #1**
```
3 → 2 → 0 → 4
    ↑_______↑   (4의 next가 인덱스 1인 노드 '2'를 가리킴)
```
tail(값 4)이 인덱스 1 노드를 가리켜 사이클이 생기므로 `true`.

**입출력 예 #2**
```
1 → 2
↑___↑   (2의 next가 인덱스 0인 노드 '1'을 가리킴)
```
tail(값 2)이 인덱스 0 노드를 가리켜 사이클이 생기므로 `true`.

**입출력 예 #3**
```
1     (pos = -1, 연결 끊김)
```
사이클이 없으므로 `false`.

## 풀이 아이디어

`solution(arr, pos)` 내부에서 (1) 연결 리스트를 구성한 뒤 (2) 사이클을 감지합니다.

### 1단계. 연결 리스트 구성
- `arr`의 각 값으로 노드를 만들고, `nodes[i].next = nodes[i + 1]`로 순서대로 연결
- `pos !== -1`이면 `nodes[tail].next = nodes[pos]`로 사이클 연결
- 빈 배열이면 사이클이 없으므로 바로 `false`

### 2단계. 사이클 감지

### 방법 1. 해시셋(Set) 활용 - O(n) 시간, O(n) 공간
- 방문한 노드를 Set에 저장
- 이미 Set에 있는 노드를 만나면 → 사이클
- 리스트 끝(`null`)에 도달하면 → 사이클 없음

### 방법 2. 플로이드 순환 감지 (Floyd's Cycle Detection) - O(n) 시간, **O(1) 공간** ⭐
- **느린 포인터(slow)**: 한 번에 1칸 이동
- **빠른 포인터(fast)**: 한 번에 2칸 이동
- 사이클이 있다면 두 포인터는 반드시 만남 (토끼와 거북이 알고리즘)
- fast가 `null`에 도달하면 사이클 없음

```
slow: 3 → 2 → 0 → 4 → 2 → ...
fast: 3 → 0 → 2 → 0 → ...
                ↑ 두 포인터가 만남!
```

## 면접 팁

- **"왜 플로이드 알고리즘인가?"**: 공간 복잡도 O(1)이 핵심. 해시셋은 O(n) 메모리가 필요하지만, 플로이드는 포인터 2개만 사용합니다.
- **"두 포인터가 반드시 만나는 이유?"**: 사이클 내에서 fast는 slow보다 1칸씩 더 빠르게 따라잡으므로, 사이클 길이 내에서 반드시 만납니다.
- **응용 질문**: "사이클 시작점을 어떻게 찾나요?" → 만난 지점에서 slow를 head로 리셋 후 둘 다 1칸씩 이동하면 사이클 시작점에서 다시 만납니다.
- **실무 연관성**: 메모리 누수 감지, 데드락 탐지, 무한 루프 방지 등에 응용됩니다.
