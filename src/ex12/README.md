# ex12. 연결 리스트 사이클 감지 (Linked List Cycle Detection)

## 문제 설명

단일 연결 리스트(singly linked list)가 주어질 때, 리스트에 **사이클(cycle)이 존재하는지** 판별하세요.

사이클이란 어떤 노드의 `next` 포인터가 이전에 방문한 노드를 가리켜, 무한 루프를 형성하는 상태를 말합니다.

사이클이 존재하면 `true`, 존재하지 않으면 `false`를 반환하세요.

> **추가 도전**: 사이클이 시작되는 노드(진입점)도 반환할 수 있나요?

## 입력 조건

- 노드의 수는 `0` 이상 `10,000` 이하입니다.
- 노드의 값(`val`)은 정수입니다.
- `pos`는 tail 노드의 `next`가 연결되는 노드의 인덱스입니다 (0-based). `-1`이면 사이클 없음.

## 예제

**예제 1 - 사이클 있음**
```
노드: 3 → 2 → 0 → 4
               ↑_______↑   (4의 next가 인덱스 1인 노드 '2'를 가리킴)

입력: head = [3, 2, 0, 4], pos = 1
출력: true
```

**예제 2 - 사이클 있음 (두 노드)**
```
노드: 1 → 2
     ↑___↑   (2의 next가 인덱스 0인 노드 '1'을 가리킴)

입력: head = [1, 2], pos = 0
출력: true
```

**예제 3 - 사이클 없음**
```
노드: 1

입력: head = [1], pos = -1
출력: false
```

## 풀이 아이디어

### 방법 1. 해시셋(Set) 활용 - O(n) 시간, O(n) 공간
- 방문한 노드를 Set에 저장
- 이미 Set에 있는 노드를 만나면 → 사이클
- 리스트 끝(`null`)에 도달하면 → 사이클 없음

### 방법 2. 플로이드 순환 감지 (Floyd's Cycle Detection) - O(n) 시간, **O(1) 공간** ⭐
- **느린 포인터(slow)**: 한 번에 1칸 이동
- **빠른 포인터(fast)**: 한 번에 2칸 이동
- 사이클이 있다면 두 포인터는 반드시 만남 (토끼와 거북이 알고리즘)
- fast가 `null`에 도달하면 사이클 없음

```
slow: 3 → 2 → 0 → 4 → 2 → ...
fast: 3 → 0 → 2 → 0 → ...
                ↑ 두 포인터가 만남!
```

## 면접 팁

- **"왜 플로이드 알고리즘인가?"**: 공간 복잡도 O(1)이 핵심. 해시셋은 O(n) 메모리가 필요하지만, 플로이드는 포인터 2개만 사용합니다.
- **"두 포인터가 반드시 만나는 이유?"**: 사이클 내에서 fast는 slow보다 1칸씩 더 빠르게 따라잡으므로, 사이클 길이 내에서 반드시 만납니다.
- **응용 질문**: "사이클 시작점을 어떻게 찾나요?" → 만난 지점에서 slow를 head로 리셋 후 둘 다 1칸씩 이동하면 사이클 시작점에서 다시 만납니다.
- **실무 연관성**: 메모리 누수 감지, 데드락 탐지, 무한 루프 방지 등에 응용됩니다.
