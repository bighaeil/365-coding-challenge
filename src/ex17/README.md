# ex17. 체육복 (Gym Suit)

## 문제 설명

점심시간에 도둑이 들어 학생들의 체육복을 훔쳐갔습니다.
다행히 여벌의 체육복이 있는 학생이 도난당한 학생에게 빌려주기로 했습니다.

단, 여벌의 체육복을 가진 학생이 체육복을 도난당했다면 다른 학생에게 빌려줄 수 없습니다.
체육복이 없으면 체육 수업을 들을 수 없고, 한 학생이 두 사람에게 빌려줄 수 없습니다.
체육복은 자기 번호의 **앞 번호** 또는 **뒷 번호** 학생에게만 빌려줄 수 있습니다.

전체 학생 수 `n`, 도난당한 학생 번호 배열 `lost`, 여벌의 체육복을 가져온 학생 번호 배열 `reserve`가 주어질 때,
체육 수업을 들을 수 있는 **학생의 최대 수**를 반환하세요.

> 프로그래머스 Level 1 / 카카오 코딩테스트 유사 유형입니다.

## 입력 조건

- 전체 학생 수 `n`은 `2` 이상 `30` 이하입니다.
- 도난당한 학생의 수는 `1` 이상 `n` 이하이며 중복 없습니다.
- 여벌의 체육복을 가져온 학생의 수는 `1` 이상 `n` 이하이며 중복 없습니다.
- 여벌의 체육복이 있는 학생만 다른 학생에게 빌려줄 수 있습니다.

## 예제

**예제 1**
```
입력: n=5, lost=[2,4], reserve=[1,3,5]
출력: 5
설명: 1번이 2번에게, 3번 또는 5번이 4번에게 빌려주면 모두 수업 가능
```

**예제 2**
```
입력: n=5, lost=[2,4], reserve=[3]
출력: 4
설명: 3번이 2번 또는 4번 중 한 명에게만 빌려줄 수 있음
```

**예제 3**
```
입력: n=3, lost=[3], reserve=[1]
출력: 2
설명: 1번과 3번은 인접하지 않아 빌려줄 수 없음
```

## 풀이 아이디어

### 핵심 아이디어 (그리디)
- 여벌이 있는 학생이 도난도 당했다면 → 자기 옷만 입고 빌려줄 수 없음 (먼저 처리!)
- 앞 번호 → 뒷 번호 순으로 빌려주면 최대 인원 확보 가능

### 방법. 그리디 — O(n)
1. `reserve`에 있으면서 `lost`에도 있는 학생 → 둘 다에서 제거 (자기 것 사용)
2. `lost`를 순회하며 `reserve`에 `lost[i]-1` 또는 `lost[i]+1` 이 있으면 빌려줌
3. 최종 수업 가능 학생 수 = `n - 남은 lost 수`

```
n=5, lost=[2,4], reserve=[1,3,5]

1단계: 교집합 처리 → 없음
2단계:
  lost[0]=2 → reserve에 1 있음 → 1이 2에게 빌려줌
  lost[1]=4 → reserve에 3 있음 → 3이 4에게 빌려줌
3단계: 남은 lost=[] → 5 - 0 = 5
```

## 면접 팁

- **"왜 그리디가 성립하나요?"** → 앞 번호부터 순서대로 처리하면, 각 여벌 학생이 최대 1명에게만 빌려줄 수 있고 인접한 학생에게만 빌려줄 수 있기 때문에 앞에서부터 욕심내도 전체 최적이 보장됩니다.
- **"여벌+도난 학생을 먼저 처리하는 이유는?"** → 이 학생이 빌려줄 수 있다고 착각하면 오답이 납니다. 먼저 제거해야 정확한 계산이 가능합니다.
- **"Set을 사용하는 이유는?"** → 빌려준 후 O(1)로 삭제하기 위해서입니다. 배열에서 삭제는 O(n)이지만 Set은 O(1)입니다.
- **카카오 실전 팁**: 그리디 문제에서는 **예외 케이스(여벌+도난 동시 보유)** 를 먼저 처리하는 것이 핵심입니다.

# 

**그리디 + Set** 풀이입니다!

---

### 🔍 동작 흐름 (`n=5, lost=[2,4], reserve=[1,3,5]`)

```
lostSet    = {2, 4}
reserveSet = {1, 3, 5}

1단계 (교집합 처리): 교집합 없음 → 변화 없음

2단계 (빌려주기):
  student=2 → reserveSet에 1 있음 → 2 해결, 1 제거
  student=4 → reserveSet에 3 있음 → 4 해결, 3 제거

lostSet = {} (비어있음)
→ 5 - 0 = 5 반환
```

### 💡 핵심 포인트

- **교집합 먼저 처리** — 여벌+도난 학생을 먼저 제거하지 않으면 잘못된 빌려주기 발생
- **Set 사용** — 삭제/조회 모두 O(1)
- **`for...of` 중 `delete`** — Set은 순회 중 삭제해도 안전
