# ex13. 이진 트리 최대 깊이 (Maximum Depth of Binary Tree)

## 문제 설명

이진 트리(Binary Tree)의 루트 노드가 주어질 때, 트리의 **최대 깊이**를 반환하세요.

최대 깊이란 루트 노드에서 가장 멀리 있는 리프 노드까지의 **노드 수**입니다.

## 입력 조건

- 노드의 수는 `0` 이상 `10,000` 이하입니다.
- 노드의 값(`val`)은 정수입니다.
- 빈 트리(`null`)의 깊이는 `0`입니다.

## 예제

**예제 1**
```
        3
       / \
      9  20
         / \
        15   7

입력: root = [3, 9, 20, null, null, 15, 7]
출력: 3
```

**예제 2**
```
    1
     \
      2

입력: root = [1, null, 2]
출력: 2
```

**예제 3**
```
입력: root = []
출력: 0
```

## 풀이 아이디어

### 방법 1. 재귀(DFS) - O(n) 시간, O(h) 공간 (h = 트리 높이)
- 현재 노드가 `null`이면 0 반환
- 왼쪽/오른쪽 서브트리의 최대 깊이를 재귀로 구함
- `1 + Math.max(왼쪽 깊이, 오른쪽 깊이)` 반환

```
maxDepth(3) = 1 + max(maxDepth(9), maxDepth(20))
            = 1 + max(1, 1 + max(maxDepth(15), maxDepth(7)))
            = 1 + max(1, 2)
            = 3
```

### 방법 2. BFS(레벨 순회) - O(n) 시간, O(n) 공간
- 큐(Queue)를 사용해 레벨 단위로 순회
- 레벨을 탐색할 때마다 깊이 카운터 +1
- 큐가 비면 최종 깊이 반환

## 면접 팁

- **재귀 vs BFS 비교**: 재귀(DFS)는 구현이 간결하지만 스택 오버플로우 위험이 있음. BFS는 메모리를 더 쓰지만 레벨별 처리가 직관적.
- **공간 복잡도 차이**: DFS는 O(h) — 균형 트리면 O(log n), 편향 트리면 O(n). BFS는 항상 O(n).
- **응용 질문**: "최소 깊이(Minimum Depth)는?"  → 리프 노드까지의 최단 경로, BFS가 더 효율적.
- **실무 연관성**: DOM 트리 탐색, 파일 시스템 깊이 계산, 조직도 분석 등에 응용.
