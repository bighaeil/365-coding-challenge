# ex23. 영단어 반복 확장 (Word Repeat Expansion)

> **ex19(숫자 문자열과 영단어)의 변형** — 영단어를 숫자로 바꾸는 것까지는 같지만, **뒤따르는 숫자는 값이 아니라 "직전 영단어를 몇 번 반복할지"** 를 뜻한다.

## 문제 설명

문자열 `s`는 **영단어**(`zero`~`nine`)와 **숫자**(`1`~`9`)가 섞여 있습니다.

- **영단어**는 대응하는 숫자로 변환합니다. (`one`→`1`, `two`→`2`, ...)
- **숫자**는 값이 아니라 **바로 앞 영단어를 그 횟수만큼 반복**하라는 의미입니다.
- 반복 숫자가 붙지 않은 영단어는 **1번**만 나타납니다.

문자열 `s`를 위 규칙으로 확장한 **결과 문자열**을 반환하세요.

각 숫자에 대응되는 영단어는 ex19와 동일합니다.

| 숫자 | 영단어 | 숫자 | 영단어 |
|------|--------|------|--------|
| 0 | zero | 5 | five |
| 1 | one  | 6 | six |
| 2 | two  | 7 | seven |
| 3 | three | 8 | eight |
| 4 | four | 9 | nine |

## 입력 조건

- `s`의 길이는 `1` 이상 `100` 이하입니다.
- 반복을 나타내는 숫자는 `1`~`9`이며, **반드시 영단어 바로 뒤에** 옵니다.
- 결과는 **문자열**로 반환합니다. (`'zero3' → '000'` 처럼 앞자리 `0`이나 반복이 그대로 보존되어야 하므로 숫자가 아닌 문자열)

## 예제

**예제 1**
```
입력: s = 'one2'
출력: '11'
설명: one=1, 2 = 앞의 '1'을 2번 반복 → '11'
```

**예제 2**
```
입력: s = 'one2two3'
출력: '11222'
설명: one=1 ×2 → '11', two=2 ×3 → '222' → '11222'
```

**예제 3**
```
입력: s = 'onetwo'
출력: '12'
설명: 반복 숫자가 없으므로 각 영단어는 1번씩 → one=1, two=2 → '12'
```

**예제 4**
```
입력: s = 'one2three'
출력: '113'
설명: one=1 ×2 → '11', three=3(반복 숫자 없음 → 1번) → '3' → '113'
```

## ex19와의 차이

| | ex19 | ex23 (이 문제) |
| --- | --- | --- |
| 영단어 | 숫자로 변환 | 숫자로 변환 (동일) |
| 뒤의 숫자 | **그 자체가 한 자리 값** (`one4` → `14`) | **직전 영단어의 반복 횟수** (`one4` → `1111`) |
| 반환 | 숫자(`Number`) | 문자열(반복·앞자리 0 보존) |

## 풀이 아이디어

### 핵심 아이디어 — "대기(pending) 버퍼" 한 칸

왼쪽에서 오른쪽으로 한 글자씩 훑으면서, **직전 영단어의 숫자를 아직 결과에 쓰지 않고 `pending`에 잠깐 보관**합니다.

- **영단어**를 만나면 → 이전 `pending`이 있으면 (반복 숫자가 안 붙었다는 뜻이므로) **1번** 기록하고, 새 영단어를 `pending`에 넣는다.
- **숫자**를 만나면 → `pending`을 그 횟수만큼 기록(`repeat`)하고 `pending`을 비운다.
- 문자열 끝에서 `pending`이 남아 있으면 **1번** 기록한다.

```
s = 'one2two3'

i=0 'one'  → pending='1'                         result=''
i=3 '2'    → result += '1'.repeat(2)='11', pending=null   result='11'
i=4 'two'  → (pending 없음) pending='2'          result='11'
i=7 '3'    → result += '2'.repeat(3)='222', pending=null  result='11222'
끝         → pending 없음                         result='11222'
```

### 영단어 매칭 팁

영단어 중 어떤 것도 다른 영단어의 접두사가 아니므로(예: `six`/`seven`, `four`/`five` 모두 겹치지 않음), 현재 위치에서 `startsWith`로 매칭되는 단어는 **항상 하나뿐**입니다. 그리디하게 매칭해도 안전합니다.

```javascript
const word = WORDS.find((w) => s.startsWith(w, i)); // 위치 i부터 매칭
i += word.length;
```

## 면접 팁

- **"왜 숫자가 아니라 문자열로 반환하나요?"** → `'zero3' → '000'`, `'nine9' → '999999999'` 처럼 앞자리 `0`과 반복 결과가 그대로 보존되어야 합니다. `Number('000')`은 `0`이 되어 정보가 사라집니다.
- **"pending 버퍼가 왜 필요한가요?"** → 영단어를 읽는 순간에는 뒤에 반복 숫자가 올지 안 올지 모릅니다. 그래서 **바로 기록하지 않고 한 칸 미뤄** 두었다가, 다음 글자가 숫자면 반복, 아니면 1번 기록으로 확정합니다.
- **"replaceAll 같은 단순 치환으로도 되나요?"** → 어렵습니다. 숫자가 "값"이 아니라 "직전 토큰에 대한 연산"이라서, 앞 토큰이 무엇인지 문맥을 기억하며 순차 파싱해야 합니다.
- **비슷한 유형** → 이 문제는 본질적으로 **런렝스 디코딩(Run-Length Decoding)** / 문자열 압축 해제입니다. `LeetCode 394. Decode String`(`'3[a]2[bc]' → 'aaabcbc'`), `LeetCode 604. Design Compressed String Iterator`가 같은 계열입니다.
