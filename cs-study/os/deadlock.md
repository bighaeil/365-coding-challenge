# 교착 상태(Deadlock)란 무엇인가요?

## 질문

교착 상태(Deadlock)가 무엇인지 설명하고, 발생 조건과 해결 방법을 말씀해주세요.

---

## 핵심 답변

### 1. 기본 개념

**교착 상태(Deadlock)** 란 둘 이상의 프로세스(또는 스레드)가 서로가 점유한 자원을 기다리면서 무한히 대기하는 상태를 말합니다.

```
프로세스 A                    프로세스 B
┌──────────┐                ┌──────────┐
│ 자원 1 보유 │ ──── 요청 ──→ │ 자원 2 보유 │
│ 자원 2 대기 │ ←── 요청 ──── │ 자원 1 대기 │
└──────────┘                └──────────┘
      ↑                          ↑
      └── 서로 기다리며 영원히 진행 불가 ──┘
```

**일상 비유**: 좁은 골목에서 두 차가 마주보고 서 있는 상황. 둘 다 상대방이 비켜주길 기다리지만, 아무도 움직이지 않습니다.

---

### 2. 교착 상태 발생의 4가지 필요 조건 (Coffman 조건)

4가지 조건이 **동시에 모두** 만족해야 교착 상태가 발생합니다.

| 조건 | 설명 | 예시 |
|------|------|------|
| **상호 배제 (Mutual Exclusion)** | 자원은 한 번에 하나의 프로세스만 사용 가능 | 프린터는 동시에 두 문서를 인쇄할 수 없음 |
| **점유 대기 (Hold and Wait)** | 자원을 보유한 채 다른 자원을 요청하며 대기 | A가 파일 잠금을 가진 채 DB 잠금을 요청 |
| **비선점 (No Preemption)** | 다른 프로세스가 보유한 자원을 강제로 빼앗을 수 없음 | OS가 프로세스의 자원을 강제 회수하지 않음 |
| **순환 대기 (Circular Wait)** | 프로세스들이 원형으로 서로의 자원을 대기 | A→B→C→A 순으로 서로 자원 요청 |

---

### 3. 교착 상태 해결 방법

#### 3-1. 예방 (Prevention) — 4가지 조건 중 하나를 제거

| 제거 대상 | 방법 |
|-----------|------|
| 상호 배제 | 자원을 공유 가능하게 만듦 (읽기 전용 자원 등) |
| 점유 대기 | 필요한 자원을 **한꺼번에** 요청하거나, 보유 자원을 반납 후 요청 |
| 비선점 | 자원을 강제 회수 가능하게 설계 |
| 순환 대기 | 자원에 **번호를 매겨** 오름차순으로만 요청하도록 강제 |

#### 3-2. 회피 (Avoidance) — 안전 상태만 허용

- **은행원 알고리즘 (Banker's Algorithm)**
  - 자원 할당 전, 할당 후에도 시스템이 **안전 상태(Safe State)** 인지 확인
  - 안전 상태: 모든 프로세스가 끝날 수 있는 실행 순서가 존재하는 상태
  - 불안전 상태면 요청을 거부

```
안전 상태 판단:
┌───────────────────────────────┐
│ 현재 가용 자원으로 완료 가능한  │
│ 프로세스가 있는가?              │
│   YES → 해당 프로세스 완료 후   │
│          자원 회수, 반복        │
│   NO  → 불안전 상태 (거부)      │
└───────────────────────────────┘
```

#### 3-3. 탐지 및 복구 (Detection & Recovery)

- 교착 상태를 허용하되, **자원 할당 그래프**로 주기적으로 탐지
- 발견 시 복구:
  - 프로세스 강제 종료 (하나씩 또는 전체)
  - 자원 선점 (일부 프로세스에서 자원 회수)

#### 3-4. 무시 (Ignore) — 타조 알고리즘

- 교착 상태 발생 확률이 매우 낮으면 아무 조치도 하지 않음
- 발생 시 수동으로 재시작
- 대부분의 범용 OS(Linux, Windows)가 채택하는 방식

---

### 4. 실전 예시: 데이터베이스 교착 상태

```sql
-- 트랜잭션 A
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;  -- row 1 잠금
UPDATE accounts SET balance = balance + 100 WHERE id = 2;  -- row 2 대기 ❌

-- 트랜잭션 B (동시에 실행)
BEGIN;
UPDATE accounts SET balance = balance - 50 WHERE id = 2;   -- row 2 잠금
UPDATE accounts SET balance = balance + 50 WHERE id = 1;   -- row 1 대기 ❌
```

**해결**: 항상 id 오름차순으로 잠금 → 순환 대기 제거

## 면접 팁

- **"교착 상태란?"** → 둘 이상의 프로세스가 서로의 자원을 기다리며 무한 대기하는 상태입니다.
- **"4가지 조건을 말해주세요"** → 상호 배제, 점유 대기, 비선점, 순환 대기. 이 4가지가 동시에 성립해야 교착 상태가 발생합니다.
- **"실제로 OS는 어떻게 처리하나요?"** → Linux, Windows 등 대부분의 범용 OS는 타조 알고리즘을 사용합니다. 교착 상태가 드물게 발생하므로 비용 대비 효율이 낮기 때문입니다. 반면 DB(MySQL, PostgreSQL)는 교착 상태 탐지 후 트랜잭션 롤백으로 자동 복구합니다.
- **"예방과 회피의 차이는?"** → 예방은 4가지 조건 중 하나를 구조적으로 불가능하게 만드는 것이고, 회피는 자원 할당 시점에 안전 상태인지 동적으로 판단하는 것입니다.
