# HTTP와 HTTPS의 차이점은 무엇인가요?

## 질문

HTTP와 HTTPS의 차이점을 설명하고, HTTPS가 어떻게 보안을 제공하는지 설명하세요.

---

## 핵심 답변

### 1. 기본 개념

| | HTTP | HTTPS |
|--|--|--|
| 풀네임 | HyperText Transfer Protocol | HTTP Secure |
| 암호화 | 없음 (평문 전송) | SSL/TLS로 암호화 |
| 포트 | 80 | 443 |
| 속도 | 빠름 | 약간 느림 (암호화 오버헤드) |
| 인증서 | 불필요 | SSL/TLS 인증서 필요 |

---

### 2. HTTPS가 보안을 제공하는 방법 — SSL/TLS 핸드셰이크

```
클라이언트                          서버
   |                                 |
   |---- ClientHello --------------->|  (지원하는 암호화 방식 목록 전송)
   |                                 |
   |<--- ServerHello + 인증서 --------|  (서버가 암호화 방식 선택 + 공개키 전송)
   |                                 |
   |  (인증서 유효성 검증)             |
   |                                 |
   |---- 대칭키 생성 & 암호화 전송 --->|  (공개키로 암호화해서 전송)
   |                                 |
   |<--- 암호화 통신 시작 ------------|  (이후 대칭키로 암호화된 데이터 교환)
```

**3가지 보안 목표:**
- **기밀성**: 데이터를 암호화해 도청 불가
- **무결성**: 데이터가 중간에 변조되지 않았음을 보장
- **인증**: 서버가 신뢰할 수 있는 곳임을 인증서로 증명

---

### 3. HTTP의 보안 취약점

```
HTTP: 클라이언트 --[아이디/비밀번호 평문]--> 서버
      중간에 패킷 가로채면 그대로 노출!

HTTPS: 클라이언트 --[암호화된 데이터]--> 서버
       가로채도 복호화 불가
```

**중간자 공격(MITM, Man-In-The-Middle Attack)**
- HTTP는 중간에 누군가 패킷을 가로채 내용을 읽거나 변조 가능
- HTTPS는 암호화로 이를 방지

---

### 4. SSL vs TLS

- SSL(Secure Sockets Layer)은 구버전, 현재는 보안 취약점으로 사용 중단
- TLS(Transport Layer Security)가 SSL의 후속 버전으로 현재 표준
- 일반적으로 "SSL"이라고 말해도 실제로는 TLS를 의미하는 경우가 많음

---

### 5. HTTPS도 완벽하지 않다

- 인증서 자체가 위조되거나 탈취된 경우 (→ 인증서 투명성 로그로 대응)
- 서버 내부에서의 데이터 처리는 암호화 범위 밖
- HTTPS는 전송 구간만 보호, 서버에 저장된 데이터는 별도 보안 필요

---

## 면접 팁

- **"HTTP는 왜 위험한가요?"** → 평문 전송이라 패킷 스니핑으로 아이디/비밀번호가 그대로 노출됩니다.
- **"SSL과 TLS의 차이는?"** → SSL은 구버전으로 현재 deprecated, TLS가 현재 표준입니다. 통상적으로 SSL이라고 부르지만 실제로는 TLS를 사용합니다.
- **"HTTPS가 느린 이유는?"** → 최초 연결 시 SSL/TLS 핸드셰이크 과정이 추가되기 때문입니다. 단, HTTP/2 이후에는 성능 차이가 거의 없습니다.
- **"대칭키와 비대칭키를 함께 쓰는 이유는?"** → 비대칭키(공개키/개인키)는 안전하지만 느리고, 대칭키는 빠르지만 키 교환이 위험합니다. 핸드셰이크에서 비대칭키로 대칭키를 안전하게 교환하고, 이후 통신은 빠른 대칭키로 처리합니다.
