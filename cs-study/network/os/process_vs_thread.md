# 프로세스와 스레드의 차이점은 무엇인가요?

## 질문

프로세스(Process)와 스레드(Thread)의 차이점을 설명하고,
각각 언제 사용하는 것이 적합한지 말씀해주세요.

---

## 핵심 답변

### 1. 기본 개념

| | 프로세스 | 스레드 |
|--|--|--|
| 정의 | 실행 중인 프로그램의 인스턴스 | 프로세스 내의 실행 단위 |
| 메모리 | 독립적인 메모리 공간 | 프로세스 메모리 공유 |
| 자원 | CPU, 메모리 등 독립적으로 할당 | 프로세스 자원 공유 |
| 통신 | IPC(파이프, 소켓 등) 필요 | 공유 메모리로 직접 통신 |
| 컨텍스트 스위칭 | 비용 큼 | 비용 작음 |
| 크래시 영향 | 다른 프로세스에 영향 없음 | 같은 프로세스 내 스레드 전체 영향 |

### 2. 메모리 구조

```
프로세스 A                   프로세스 B

├──────────────┤            ├──────────────┤

├──────────────┤            ├──────────────┤
│   스택 영역   │            │   스택 영역   │
└──────────────┘            └──────────────┘
     완전 독립                   완전 독립


프로세스 내 멀티스레드
┌─────────────────────────────────┐

```

### 3. 각각 언제 사용할까?

**멀티프로세스가 적합한 경우**
- 크래시 격리가 중요할 때 (한 프로세스가 죽어도 다른 프로세스에 영향 없음)
- 보안이 중요할 때 (메모리 공간이 완전히 분리)
- 예: 크롬 브라우저 (탭마다 별도 프로세스)

**멀티스레드가 적합한 경우**
- 빠른 데이터 공유가 필요할 때
- 컨텍스트 스위칭 비용을 줄이고 싶을 때
- 예: 웹 서버 (요청마다 스레드 생성), 게임 엔진

---

### 4. 컨텍스트 스위칭

CPU가 실행 중인 작업을 바꿀 때 현재 상태(레지스터, 프로그램 카운터 등)를 저장하고 다음 작업의 상태를 복원하는 과정입니다.

- **프로세스 컨텍스트 스위칭**: 메모리 주소 공간까지 교체 → 비용 큼
- **스레드 컨텍스트 스위칭**: 스택과 레지스터만 교체 → 비용 작음

## 면접 팁

- **"스레드는 무엇을 공유하고, 무엇을 독립적으로 가지나요?"** → 코드/데이터/힙은 공유, 스택과 레지스터는 독립적으로 가집니다.
- **"멀티스레드의 단점은?"** → 공유 메모리로 인한 동시성 문제(race condition)가 발생할 수 있어 동기화(뮤텍스, 세마포어)가 필요합니다.
- **"크롬이 탭마다 프로세스를 쓰는 이유는?"** → 탭 하나가 크래시 나도 다른 탭에 영향을 주지 않기 위해서입니다. 격리가 핵심입니다.
- **Node.js 연관**: Node.js는 싱글 스레드 기반이지만 Worker Threads로 멀티스레딩을 지원합니다. I/O 작업은 이벤트 루프로 비동기 처리합니다.
